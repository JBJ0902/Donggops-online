# v31 로컬 계정/사용자 경쟁전 패치

## GitHub Pages 단독 구현 범위
- GitHub Pages는 정적 파일 호스팅 구조이므로 서버 DB 없이 `localStorage` 기반으로 구현했습니다.
- 계정, 키 설정, 기록, 경쟁전 랭킹은 같은 브라우저/기기 안에서 저장됩니다.
- 전 사용자 공용 서버 랭킹/진짜 보안 로그인은 Supabase/Firebase/서버 API가 필요합니다.

## 추가/수정 내용
- 메인 타이틀 우측 상단 LOGIN 버튼 추가
  - 마우스 오버 시 계정생성/로그인 메뉴 펼침
  - 로그인 후 내 기록/로그아웃 메뉴 표시
- ID/PW/닉네임 로컬 계정 생성
  - ID: 영문/숫자 4~18자리
  - PW: 영문 대문자/소문자/숫자 조합 10~18자리
  - 닉네임: 한글 12자 또는 영문 18자 이내
- 로그인 닉네임이 플레이어 이름으로 표시되도록 적용
- 로그인 유저별 키 설정 분리 저장
- 내 기록 페이지 추가
  - 플레이/승리/최고점수/최고CPM/최대콤보/최고스테이지
  - 최근 기록 목록 스크롤 지원
- 동꼽즈 게임 모드에 `C : 사용자 경쟁전` 추가
  - 로그인한 사용자만 진입
  - 기존 1~5스테이지 + 신규 6~10스테이지 구성
  - 6~10스테이지 난이도: 초고수 / 프로 / 마스터 / 레전드 / 동꼽신
  - 5스테이지보다 높은 AI CPM과 더 강한 밸런스로 조정
  - 패배 시 즉시 경쟁전 종료 후 랭킹 화면으로 이동
- 사용자 경쟁전 최종 랭킹 화면 추가
  - 총점 / 도달 스테이지 / 최고CPM / 최대콤보 기준 정렬
  - 1위~100위까지 마우스 휠 스크롤 표시
- 6~10스테이지 BGM은 추후 파일 추가 시 확장 가능하도록 stage5 BGM fallback 구조 유지

## 업로드 방법
- 전체 덮어쓰기: 압축 해제 후 전체 파일 업로드
- 최소 덮어쓰기: `game.js`와 이 패치노트만 업로드
